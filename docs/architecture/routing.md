# Routing & Navigation

## Router: GoRouter

Paisa uses **GoRouter** (v13) with **type-safe routes** generated by `go_router_builder`. Navigation is URL-based and works identically on mobile (deep links) and web (browser address bar).

## Route Tree

```
/intro                          → IntroPage (welcome screen)
/onboarding                     → UserOnboardingPage (name, photo, currency)
/biometric                      → BiometricPage (auth gate)
/landing                        → LandingPage (main shell)
  /landing/search               → SearchPage
  /landing/debit                → DebitPage (add/edit debt)
  /landing/transaction          → TransactionPage (add/edit transaction)
  /landing/account              → AccountPage (add/edit account)
  /landing/recurring            → AddRecurringPage
  /landing/export               → ExportAndImportPage
  /landing/settings             → SettingsPage
    /landing/settings/font-picker
    /landing/settings/language-picker
  /landing/category             → CategoryPage (add/edit category)
    /landing/category/icon-picker
  /landing/transactions/:categoryId → TransactionByCategoryPage
```

## Route Classes

Each route is a `GoRouteData` subclass annotated with `@TypedGoRoute`. The `build()` method returns the page widget:

```dart
@TypedGoRoute<TransactionPageData>(path: 'transaction')
class TransactionPageData extends GoRouteData {
  const TransactionPageData({
    this.accountId,
    this.categoryId,
    this.transactionId,
    this.transactionType,
  });

  final int? accountId;
  final int? categoryId;
  final int? transactionId;
  final TransactionType? transactionType;

  @override
  Widget build(BuildContext context, GoRouterState state) {
    return TransactionPage(
      transactionType: transactionType,
      accountId: accountId,
      categoryId: categoryId,
      transactionId: transactionId,
    );
  }
}
```

Navigate using typed constructors — no magic strings:

```dart
// Navigate to add transaction as expense
TransactionPageData(transactionType: TransactionType.expense).go(context);

// Navigate to edit an existing transaction
TransactionPageData(transactionId: 42).go(context);

// Navigate to settings
const SettingsPageData().go(context);
```

## Redirect Logic (Auth & Onboarding Guard)

GoRouter's `redirect` function runs on every navigation and acts as the app's gate keeper. It checks Hive settings to decide where the user should be:

```mermaid
flowchart TD
    START([App Launch]) --> CHECK1{Intro done?}
    CHECK1 -- No --> INTRO[/intro]
    CHECK1 -- Yes --> CHECK2{Name & image set?}
    CHECK2 -- No --> ONBOARD[/onboarding]
    CHECK2 -- Yes --> CHECK3{Country selected?}
    CHECK3 -- No --> ONBOARD2[/onboarding?forceCountrySelector=true]
    CHECK3 -- Yes --> CHECK4{Biometric enabled?}
    CHECK4 -- Yes --> BIO[/biometric]
    CHECK4 -- No --> LAND[/landing]
```

The router is wired to Hive's `listenable()`, so when a settings key changes (e.g., user completes onboarding), the router automatically re-runs the redirect and navigates to the next screen — no manual `context.go()` needed.

```dart
final GoRouter goRouter = GoRouter(
  refreshListenable: settings.listenable(keys: [
    userIntroFinishedKey,
    userNameSetKey,
    userImageKey,
    userCategorySelectorKey,
    userAccountSelectorKey,
    userCountryKey,
    userAuthKey,
  ]),
  redirect: (context, state) {
    // See routes.dart for full implementation
  },
);
```

## Navigation Patterns

### Pop (back navigation)
```dart
context.pop();
```

### Push to named route
```dart
const SearchPageData().go(context);
```

### Push with params
```dart
AccountPageData(accountId: account.superId).go(context);
```

### Replace current route
```dart
const LandingPageData().go(context);
```

## Deep Link Support

Because GoRouter uses URL-based routing, deep links are supported on all platforms:

- **Android**: Intent filter in `AndroidManifest.xml` for `paisa://` scheme
- **Web**: Full URL routing in the browser (no hash-based URLs thanks to `url_strategy`)
- **iOS**: Universal Links or custom scheme

## Web URL Strategy

The app uses `url_strategy` to remove the `#` from web URLs:

```dart
// lib/main.dart
usePathUrlStrategy(); // Called before runApp()
```

This means web URLs look like `/landing/search` instead of `/#/landing/search`.
